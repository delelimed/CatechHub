1. Architettura del bootstrap

L'applicazione esegue un bootstrap sequenziale in 6 fasi (main.dart), ciascuna con initializzazione bloccante e propagazione errori. La funzione main() coordina il flusso tramite AsyncNotifierProvider:

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await runZonedGuarded(() async {
    final bootstrap = BootstrapService();
    await bootstrap.phase0();  // Binding Flutter
    await bootstrap.phase1();  // Servizi Dart (Clock, Logger, PathProvider)
    await bootstrap.phase2();  // Hive init + auto-recovery
    await bootstrap.phase3();  // Apertura Box con corruption recovery
    await bootstrap.phase4();  // Analytics / Privacy
    await bootstrap.phase5();  // Servizi aggiornamenti
    await bootstrap.phase6();  // runApp
  }, _fatalErrorHandler);
}

3.4 HKDF — Session Key derivation

final HKDF_INFO = 'CatechHub_SessionKey_v2';
// info = HKDF_INFO + deviceIdA + deviceIdB (UTF-8 encoded)

Dopo lo scambio ECDH, il shared secret viene derivato tramite HKDF con SHA-256 per produrre la chiave di sessione AES-256-GCM. L'info string include gli ID dei due dispositivi per garantire chiavi uniche per coppia di dispositivi.

3.5 Legacy CBC fallback

Per compatibilità con i pacchetti v1 (prime versioni dell'app), EncryptionService implementa un fallback su AES-256-CBC con padding PKCS7. Il campo "alg": "AES-256-CBC" nel JSON del pacchetto determina l'uso del fallback.

3.6 Formato del pacchetto cifrato

{
  "v": 2,               // versione formato
  "kdf": "PBKDF2",      // key derivation function
  "iter": 210000,       // iterazioni PBKDF2
  "alg": "AES-256-GCM", // algoritmo cifratura
  "salt": "aGVsbG8=",   // salt base64
  "nonce": "d29ybGQ=",  // nonce base64
  "data": "Y2l0ZXJv"    // ciphertext + tag base64
}

5. Security architecture

5.1 freeRASP

Libreria di runtime application self-protection (RASP) che rileva 8 categorie di minacce:

CategoriaRilevamento
Root / Magisk / KernelSUSuperuser binary, mount namespace, SELinux
EmulatorBuild fingerprint, QEMU drivers, Goldfish CPU
App IntegrityFirma APK modificata, debug build
Hooking / Frida / XposedLibrerie di hooking, classLoader injection
Device BindingGoogle SafetyNet / Play Integrity attestation
ADBUSB debugging enabled
DebuggerJDWP attached, debug flag
Developer OptionsImpostazioni sviluppatore attive

Configurazione: killOnBypass = false (l'utente riceve una schermata di blocco personalizzata SecurityBlockScreen ma non viene forzato alla chiusura).

5.2 Auth — local_auth

Authentication auth = Authentication();
bool canAuth = await auth.canCheckBiometrics || await auth.isDeviceSupported();
// biometricOnly = false → fallback a PIN/Pattern/Password del dispositivo

La sessione vive esclusivamente in RAM: nessun token persistito. Il timeout background è di 120 secondi. Allo scadere, la sessione viene invalidata e l'utente deve riautenticarsi.

5.3 Screen security

// MainActivity.kt
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.ICE_CREAM_SANDWICH) {
  window.setFlags(
    WindowManager.LayoutParams.FLAG_SECURE,
    WindowManager.LayoutParams.FLAG_SECURE

FLAG_SECURE blocca screenshot, screen recording e la visualizzazione nelle recenti apps.
